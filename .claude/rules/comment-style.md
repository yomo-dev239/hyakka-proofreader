# コメント規約

コード内のコメントは以下のルールに従う。

## 1. 1 行コメント

- `//` で記述する
- **句読点（。、）をつけない**。文の途中・末尾どちらも不可
- 句読点なしで成立する 1 文にする。区切りが必要なら読点の代わりに `（）` を使うか、文を組み替える

```ts
// ✗ 悪い例
// 1 チャンクの目安文字数。env CHUNK_CHARS で上書き可能
// テキストを校正し、観点別の指摘を返す

// ✓ 良い例
// env CHUNK_CHARS で上書きできる 1 チャンクの目安文字数
// テキストを校正して観点別の指摘を返す
```

## 2. 2 行コメント

- 2 行に分けず、1 行に収める（収めたうえで「1 行コメント」のルールに従う）

## 3. 3 行以上のコメント

- `/** */` で記述する
- 1 行目と 2 行目の間を 1 行空ける
- **1 行目はタイトルもしくは概要**。句読点をつけない
- 2 行目以降は詳細説明。通常どおり句読点を使ってよい

```ts
/**
 * 長文を校正用のチャンクに分割する
 *
 * 行（改行）を最小単位として貪欲に詰め、文の途中では切らない。
 * 1 行が長すぎる場合のみ、文末（。！？）→ ハードカットの順でさらに分割する。
 */
```

## 4. doc コメント（宣言の説明）

- `/** */` のまま記述する（`//` に変換しない）
- 1 行で書けるものは 1 行。その場合は「1 行コメント」と同様に句読点をつけない
- 3 行以上になる場合は「3. 3 行以上のコメント」のフォーマットに従う

```ts
/** 各観点の判定基準（LLM に渡してブレを抑える） */
const PERSPECTIVE_GUIDE = `...`;
```
